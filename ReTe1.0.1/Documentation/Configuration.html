The file Configuration.txt


This is a parameter file in Retrotector/Database. It contains information used at startup. Here is an example of its contents:

WorkingDirectory: /Users/gsper/RetroTector10/Workplace
Executormenu::
 SweepDNA
 SweepScripts
 Genomeview
 CollectGenome
 Sequenceview
 CollectMotifHits
 RenameFiles
::
ORFIDdefaults::
 MinSSScore: 700
::
LTRIDdefaults::
 SingleLTRThreshold: 0.185
::

There are two parameters that should always be defined:

  • WorkingDirectory: a singleparameter specifying the directory of DNA files and results
  • Executormenu: a multiparameter partly or wholly specifying the look of the Execute menu of the RetroTector window. The menu will begin as specified, (except lines whose corresponding class cannot be found either in RetroTector.jar or in the 'plugins' directory). Executor classes not included here will appear lower down in the menu, typically in alphabetical order.
The default values of parameters of Executors, Motif classes, Brooms and ChainProcessors may be set using multiparameters as can be seen above. The leading blanks in the lines inside the parameters are not required but enhance readability. The executor ORFID has a parameter 'MinSSScore', whose normal default value is 600, but will, using this Configuration.txt, be set to 700 at startup. Similarly, the executor LTRID has (among many others) the parameter 'SingleLTRThreshold' which will receive a new default value.

There is nothing wrong with setting a default value to the same as the one already existing, just to make sure.

The parameters available for the various Executors, Motif classes, Brooms and ChainProcessorsare documented with them. In the case of the Executors, they may also be inspected after starting them from the Execute menu.

A line 'ClusterMode: Yes' may improve efficiency when running on multiple processors, see main page.
