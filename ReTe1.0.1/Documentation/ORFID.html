ORFID


ORFID is an Executor which constructs suggested proteins, "puteins", coded by ERVs found by RetroVID. It depends on information about Motif hits found by RetroVID and transferred through an ORFID script generated by RetroVID, and it is therefore not normally meaningful to run it interactively.

An ordinary ORFID script specifies which Gene and which virus genus are applicable, and ORFID chooses an Alignment file accordingly. The putein is constructed in quite close agreement with the DNA; all ORFID does is occasionally to skip one or two bases to cause a frame shift, and suggest where the putein starts and ends.

For ORFID to work, at least one of the Motif hits provided by RetroVID must belong to a Motif that can be located in the Alignment. The putein will be constructed in at least two segments; one leading (up to the first located Motif hit); one trailing (after the last located Motif hit); and possibly more if there are more than one located Motif hit.

Within each segment, the putein is constructed so as to maximize a score, which for each acid is either (depending on which is larger):
  • An 'aligned' score (max 1.0) dependent on the similarity (according to the AcidMatrix) between the acid and the acid suggested by the Alignment at this position.
  • An 'unaligned' score (max 0.85) indicating how likely the reading frame of the acid is to be a coding frame; it is calculated from several markers with variable weigths (see below).
In the case of the leading and trailing segments, a number of other criteria related to the end points are also considered, such as
  • score of PuteinStartMotif
  • score of PuteinEndMotif
  • similarity to Kozak consensus
  • proximity to hits by SpliceAcceptorMotif
  • proximity to hits by SiSeqMotif
  • proximity to hits by ProteaseCleavageMotif
  • proximity to hits by SlipperyMotif
  • proximity to hits by PseudoKnotMotif
  • stop codons
The start and end points of the putein yielding the best scores according to these criteria are chosen.

RetroVID provides ORFID with a list of all Motif hits in the relevant Gene, in the parameter HitInfo. They may, however, not all be useful to ORFID. In particular, hits with a low score may be false, which is not a big deal for RetroVID, but may reduce ORFID to gibberish. Therefore three parameters (ScoreFactor, MinSSScore and MinHitScore) are related to weeding out of such hits. In principle, if the sum of the squared hit score is less than MinSSScore, ORFID exits. If any Motif hit has a score lower than MinHitScore or lower than the average score times ScoreFactor, it is not used. If after this, no hit by a Motif in a MotifGroup with an anchor in the Alignment is left, ORFID exits.

Normally, RetroVID generates the ordinary ORFID scripts unsweepable (suffix 'Script.txt') and an ORFID "superscript", which lists the ordinary scripts. It has a name without Gene specification and suffix 'MScript_.txt' or 'NScript.txt'. When it is executed, ORFID generates the puteins in parallell, so that their ends are adjusted to each other according to a complex set of criteria.

Under unfavorable circumstances, in particular when Pol contains Motif hits only in RT, ORFID may run out of memory, resulting in a OutOfMemoryError message and/or an empty Putein file

The output of ORFID consists of:
  • A Putein file, ie a Puteinview script describing the putein, the Motif hits used in constructing it, the reading frame and some indications about probable start and end of the putein.
  • Occasionally, a second Putein file with the same name as the above, with an 'S' (for 'straightened') interspersed. It will be similar to the above, but with a seemingly unnecessary deviation from an ORF (which may or may not be due to imperfections in the Alignment file) removed.
  • (Only if the Debugging option is on) An ORFIDview script (with name beginning ORFIDout) with more detailed information, mainly useful when the putein looks all screwy. The Putein file will contain information about all the possible leading and trailing putein segments not used. This may possibily lead to an OutOfMemoryError.

The parameters of ORFID are:
  • InputFile: Name of the file to get parameters from. Only used in interactive mode.
  • GeneScripts: Only if the script is a "superscript" (see above). A list of ordinary scripts.
  • Database: Database subdirectory to use.
    Default: Ordinary.
  • Eight parameters indicating the weighting when calculating 'unaligned' score:
    • NonAlignedScore: The initial value assigned to the score for unaligned fitting. Default: 0.4.
    • StopCodonFactor: Factor to multiply stop codon load by in unaligned fitting. Default: -0.4.
    • FrameShiftFactor: Penalty for frame shift in unaligned fitting. Default: -1.5.
    • ORFHexamerFactor: Factor to multiply ORFHexamerModifier by in unaligned fitting. Default: 0.2.
    • NonORFHexamerFactor: Factor to multiply NonORFHexamerModifier by in unaligned fitting. Default: -0.1.
    • GlycosylationFactor: Factor to multiply GlycSiteModifier by in unaligned fitting of env. Default: 0.2.
    • InORFBonus: Bonus to add if an acid is in a long (>50 acids) ORF. Default: 0.1.
    • MotifHitFactor: Factor to multiply score of Motif hits by in unaligned fitting. Default: 0.2

  • MasterSkipPenalty. Penalty for skipping a filled column in alignment. Default: -0.2
  • StopCodonValue. Value of stop codon in aligned fitting. Default: -15.
  • ScoreFactor: Factor for discarding lowscoring MotifHits. Default: 0.1.
  • MinSSScore: Lowest acceptable sum of squared MotifHit scores.
    Default: 600.
  • MinHitScore: Lowest acceptable individual MotifHit score.
    Default: 15.
  • Gene: gag, pro, pol or env.
  • VirusGenus: Any virus genus symbol.
  • DNAFile: The name of the file to read DNA from. Default: Directory name + .txt
  • Strand: Primary or Secondary. Default: Primary.
  • FirstDNAStart: Calculated earliest possible start of putein in DNA. Default: 1.
  • LastDNAStart: Calculated latest possible start of putein in DNA. Default: 1.
  • FirstDNAEnd: Calculated earliest possible end of putein in DNA. Default: 200000.
  • LastDNAEnd: Calculated latest possible end of putein in DNA. Default: 200000.
  • HitInfo: A list of Motif hits. Each line contains:
    • Hotspot position of the hit.
    • The score of the hit.
    • The Motif group of he Motif.
    • The identifying string of the Motif class.
    • The virus genus of the Motif.
    • A string, typically the MOTIFSTRING of the Motif, with the hotspot marked with '#'.
    • The MOTIFORIGIN of the Motif.
  • OutputFile: Name of file for detailed output (ORFIDview script). Constructed from name of script or input file.
  • PuteinFile: Name of file for putein output (Puteinview script). Constructed from name of script or input file.
  • ChainNumber: Number of parent Chain.
  • Debugging: If this is Yes, a lot of extra information is written to Standard Output, and an ORFIDview script is generated. Default: No.
  • Trouble: If present in the script, it describes some problem (eg BrokenChain) encountered by RetroVID, suggesting that the output of ORFID may not be meaningful. It is copied to the putein file.
