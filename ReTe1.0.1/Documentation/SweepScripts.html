SweepScripts

This Executor handles batch execution.

Unlike most other Executors, SweepScripts does not show a parameter window when started interactively, but rather goes right ahead. All the information required is the current Working directory.

In executing, SweepScripts goes through the Working directory and all directories inside it, searching for sweepable scripts. Whenever one is encountered, it will be executed. After execution, its file name will be changed so that it is no longer a sweepable script, ie the terminal 'Script_.txt' is changed into 'Script.txt'.

If new sweepable scripts are created in this process, they will also be found and executed.

You should not depend on SweepScripts executing the sweepable scripts in any particular order.

If a script was not successfully executed because of an internal RetroTector error, SweepScripts will normally continue with the next script. Before exiting, a list of such unprocessed scripts will be presented.

If a Java runtime error occurs when executing a script, SweepScripts will normally abort.

There are two ways to interrupt execution of SweepScripts from the 'Control' menu of the RetroTector window. The Abort command will terminate abruptly (though not always instantaneously) which may leave files half finished. The 'Stop SweepScripts' command will terminate after allowing the current script to be executed in good order.
