RetroTector

 

CheckDatabase

This Executor in plugins is a utility used to check that a subdirectory of  RetroTector/Database is correct, which it may not be after editing; there are a number of pitfalls.

CheckDatabase is usually invoked interactively. There is one parameter:

  • 'Database': the name of the Database subdirectory to use. Default is 'Ordinary'.
CheckDatabase does roughly the following (errors are normally output to the text window and to LatestErrorLog.txt):
  • All files are checked that they contain 'Last changed yy mm dd', and that the date specified agrees with the date the file was actually last changed. If not, a 'Modification date mismatch:' error is output.
  • All files with names ending in 'Alignment.txt' are checked:
    1. That all MotifGroup anchors correspond to actual MotifGroups, or an 'Unrecognised MotifGroup name' is reported.
    2. That the distances between the MotifGroup anchors (not calculated using sections marked Suspect) are consistent with the distance ranges specified in Motifs.txt, or an 'outside' error is reported.
  • SubGenes.txt is checked for symmetry of the distance ranges, or a 'Distance mismatch' error is reported.
  • MotifGroups.txt is checked for internal consistency of distances, or an 'Invalid distance' or 'Distance mismatch' error is reported.
These messages in the text window are really objections rather than error messages. You may or may not take them to heart. There may also be genuine 'red' errors, usually because of syntax errors in the files. These are very likely to recur when the files are put to use, and must be taken seriously.
