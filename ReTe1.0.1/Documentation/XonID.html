XonID


XonID is an Executor which searches for traces of open reading frames not identified by ORFID, avoiding the reading frame of a parallell Putein. It uses methods similar to ORFID, but without use of Alignments. There are two strata of findings:

  • XXons: these begin at a SpliceAcceptorMotif hit or at a methionine codon at most 200 bases after one, and end at a stop codon or a SpliceDonorMotif hit. XXons have a reasonable chance of actually corresponding to exons.
  • Xons: these have no particular requirements except that they should have a low content of stop codons and frame shifts, and avoid the reading frame of a parallell putein or XXon. Xons are very vague entities which should not be taken seriously in the absence of corroborating evidence, but may sometimes give useful hints.
XonID is normally run using a script generated by RetroVID. It will not run if there are any unswept ORFID scripts in the directory.

The output of XonID consists of a number of XXon- and Xon- files. These are essentially Puteinview scripts, but without a number of not meaningful parameters.


The parameters of XonID are:
  • InputFile: Name of the file to get parameters from. Only used in interactive mode.
  • MinChainScore: If the parent Chain has lower score than this, nothing is done. Default: 1000.
  • LTRGagMinGap: If the gap between 5'LTR and Gag is at least this much, XonID acts. Default: 100.
  • GagProMinGap: If the gap between Gag and Pro is at least this much, XonID acts. Default: 100.
  • ProPolMinGap: If the gap between Pro and Pol is at least this much, XonID acts. Default: 100.
  • PolEnvMinGap: If the gap between Pol and Env is at least this much, XonID acts. Default: 100.
  • EnvLTRMinGap: If the gap between Env and 3'LTR is at least this much, XonID acts. Default: 100.
  • Database: Database subdirectory to use.
    Default: Ordinary.
  • ChainNumber: Number of parent Chain.
  • FaultsPer51: A parameter regulating the fault tolerance of the algorithm, mainly for XXons. The higher FaultsPer51, the more tolerant. Default: 12.
  • SnakeThreshold: A parameter regulating the fault tolerance of the algorithm for Xons. The higher SnakeThreshold, the more tolerant. Default: 0.2.
  • ChainsFile: Name of SelectedChains file to use.
    Default: 001SelectedChains.txt
  • Five parameters with similar roles as in ORFID:
    • NonAlignedScore: The initial value assigned to the score for unaligned fitting. Default: 0.4.
    • StopCodonFactor: Factor to multiply stop codon load by in unaligned fitting. Default: -0.4.
    • FrameShiftFactor: Penalty for frame shift in unaligned fitting. Default: -1.5.
    • ORFHexamerFactor: Factor to multiply ORFHexamerModifier by in unaligned fitting. Default: 0.2.
    • NonORFHexamerFactor: Factor to multiply NonORFHexamerModifier by in unaligned fitting. Default: -0.1.

  • DNAFile: The name of the file to read DNA from. Default: Directory name + .txt
  • Strand: Primary or Secondary. Default: Primary.
  • ChainStart: First position of search area in DNA. Default: 1.
  • ChainEnd: Last position of search area in DNA. 200000.
