Brooms


Brooms are Java classes to be included as needed in the plugins directory. They are used by SweepDNA to remove elements (mainly transposons) which may interfere with the later analysis.

All Brooms implemented so far are subclasses of StandardBroom. A StandardBroom is built around a consensus base sequence and (optionally) a variable length poly-A tail. Given a consensus, a custom StandardBroom is very easy to program, and may be requested from the author.

A Standard Broom selects a number of equidistant n-mers from the consensus and searches for identical sequences in the DNA. If one is found, it is extended by a moderately fault-tolerant dynamic programming algorithm (similar to that used by LTRID to compare LTR candidates) and if it is thus extended over at least 100 positions it may be extended with a poly-A tail, and marked as an insert in the DNA file.

All Brooms have the parameter:
  • BroomPriority. Brooms are applied in descending order of this.
A StandardBroom has the following parameters:
  • MerSize: The length of the n-mers in the initial search.
  • MerStep: The distance between n-mers in the consensus.
  • Tolerance: A measure of the fault tolerance in the dynamic programming algorithm.
  • TailMinLength (if poly-A tail is included): The shortest acceptable distance between consecutive non-A in the tail.
So far, these StandardBrooms have been implemented:

For primate genomes:
  • L1Broom, which is included in RetroTector.jar and is applied by SweepDNA by default, to search for LINE L1 fragments.
For other genomes, with names directly related to the names of the respective transposons:

    For the canine genome:
  • SINEC_CfBroom
    For the chicken genome:
  • CR1_BBroom
  • CR1_B2Broom
  • CR1_FBroom
  • CR1_YBroom
    For the mouse genome:
  • B1Broom
  • B2Broom
  • L1_MMBroom
  • L1MA9_5Broom
  • L1MC3Broom
  • L1MDA_5Broom
  • L1MEC_5Broom
  • L1P_MA2Broom
  • L2ABroom
