Motifs.txt


Motifs.txt is a file in Database defining

Motifs

Motifs are operators that at a particular position (the target position) in a DNA sequence detect the presence of some quantitative feature to a degree in excess of a threshold, or the presence of an all-or-nothing feature. When such is detected, a 'Motif hit' is registered, with a score and a hotspot at the target position.

A Motif type is defined either by a class inside RetroTector.jar (belonging to the package 'builtins') or by a .class file in the plugins directory. Individual Motifs are created at RetroTector startup, using the Motifs.txt file which provides individual details. Each Motif class has a unique identifying string which is utilized in the second column of Motifs.txt.

Most Motif types belong to the subclass OrdinaryMotif. An OrdinaryMotif can calculate a raw score at any position in a DNA sequence, and yields no more than one hit in the same position. If the raw score is below a certain threshold, no hit is registered. If the raw score is above the threshold, a Motif hit is registered, with a final score proportional to the difference between the raw score and the threshold. Usually the proportionality factor is chosen in such a way that if the raw score equals its theoretical maximum, the final score equals the maximal score defined in Motifs.txt. The threshold is usually adjusted according to the character of the DNA sequence to which it will be applied, by calculating the raw score in 1000 positions distributed along the sequence. The threshold is then set to the average of these scores plus their standard deviation times a factor (typically 5.5).

Therefore, in addition to its hitfinding capacity, a Motif class has a 'refresh' function, applied before the hitfinding, which calculates the threshold, and/or possibly does other preparations.

Some properties of Motif classes may also be determined by named parameters, whose default value may be changed through Configuration.txt . For example, the PPTMotif has a parameter named 'LowLimit' usually with the value 20. If you want to try 22 instead, you include a parameter like this

PPTMotifdefaults::
 LowLimit: 22
::

in Configuration.txt.

At startup, all the classes in plugins are absorbed into RetroTector. It is therefore possible to write your own Motif classes and drop them into the plugins directory. See Programming on your own .

    • HyPhobMotif
    • PPTMotif

    • PuteinStartMotif
    • RMotif

    • SlipperyMotif

    • PseudoKnotMotif
    • SingleAcidMotif

Structure of Motifs.txt

Motifs.txt is a parameter file. The bulk of the Motifs are specified in the parameter Motifs. Specialized Motifs are contained in the parameters LTR1Motifs, LTR2Motifs, ComponentMotifs, KozakMotif, SpliceAcceptorMotif, SpliceDonorMotif, SlipperyMotif, ProteaseCleavageMotif, PseudoKnotMotif and possibly others. A Motif is defined by a single line in any of these, with a common structure. Unlike in many other files in Database, the fields are not space-delimited but rather contiguous, with fixed width. They are as follows (with the names used in the Java source):
  • MOTIFID (width 11 characters). An integer, preferably unique. Its main use is to identify the Motif in error messages, in CollectMotifHits and in forming CoincidenceMotifs.
  • MOTIFTYPE (5 characters). The identifying string of the Motif type to which the Motif belongs.
  • MOTIFLEVEL (2 characters). At present, either 'D', indicating a detection Motif, ie one used in the detection process, or 'C' for a characterization Motif, used only in refining already detected chains.
  • MOTIFRVGENUS (25 characters). One or more of the identifying characters of virus genera in which this Motif occurs.
  • MOTIFSDFACTOR (5 characters). This field may be used in any way that suits the particular Motif class, but primarily to redefine the factor used in calculating raw score threshold.
  • MOTIFSUBGENE (5 characters). Name of the SubGene to which this Motif belongs, or '????' if not defined.
  • MOTIFGROUP (5 characters). Name of the MotifGroup to which this Motif belongs, or '????' if not defined.
  • MAXSCORE (5 characters). The maximal score allowed for hits with this Motif.
  • PARAM (5 characters). This field may be used in any way that suits the particular Motif class, but primarily to specify raw score threshold (in OrdinaryMotifs). If there is a real number in this field, the procedure described above for calculating raw score threshold is omitted and the value is used instead.
  • MOTIFSTRING  (55 characters). Contents vary according to Motif class, but define what feature the Motif scores.
  • MOTIFORIGIN (arbitrary length). A comment. If you use the CollectGenome command, make sure that for all PBS Motifs, MOTIFORIGIN contains a specification of the relevant tRNA, of the form tRNAxxx (case irrelevant) where the xxx is a three-letter amino acid abbreviation.
