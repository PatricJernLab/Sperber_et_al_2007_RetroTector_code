Parameter file format


Most of the text files handled by RetroTector follow this format.

A parameter file consists of three types of components, which may be present in any number and order:

  • Multiparameters. Such a one consists of
    • The key line, consisting of
      • The key, a character string (preferably without blanks or other esoteric characters)
      • A double colon ('::')
    • Contents, any number of lines of text, except one that is identical to
    • The closing line, consisting only of a double colon ('::')
  • Singleparameters, consisting of a single line with constituents
    • The key, as above
    • A single colon
    • An optional blank (or more)
    • Contents, any character string not beginning with a blank
  • Comments, beginning with a '{' character. Please note that such a line inside a multiparameter is part of it, not a comment.
In general, singleparameters are used to specify parameter values, multiparameters for data.


A special type of parameter file is the
executor script

It is a parameter file beginning with a singleparameter with key 'Executor' and contents the name of an executor. Executor scripts may be executed either from the RetroTector window or automatically by SweepScripts. This means that the executor in question is executed and the rest of the file is given to it to supply it with parameters.


A special type of executor script is the
sweepable script. This is just an executor script internally, but its file name ends with 'Script_.txt'. This marks it as a target for automatic execution by SweepScripts, which will execute it and remove the underscore from the file name.
