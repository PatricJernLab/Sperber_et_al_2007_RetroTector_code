The LatestErrorLog_....txt files


These are numbered text files in the Database directory where messages displayed in the RetroTector window are also stored. A new one is created each time RetroTector starts up and closed on exit, though the closing behaviour in connection with errors etc is somewhat implementation dependent. The error log file will be properly closed only if you terminate RetroTector with the Quit menu command, not through closing the text window or something.

A log file may receive a different name if it is specified as an L: parameter in the starting command, see RetroTector.html.

SweepScripts adds a line for each Executor script it executes, making it easy to see which one caused the error message.
