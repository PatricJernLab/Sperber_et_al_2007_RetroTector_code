Generate your own Database subdirectory

Let's say you are not satisfied with the available Database. Perhaps you want to extend the use of RetroTector to a new retrovirus genus. You may then make a new subdirectory in Database. It is laborious but possible. The only tool needed is a simple text editor that can, and does, generate simple text files (Word .doc files, for example contain hidden code bytes and are banished).

The sequence of operations (which is likely to be iterated a lot of times) is this:

  1. Make a new Motifs.txt file - probably by editing a copy of an old one.
  2. Similarly make new Alignment files. Take care with the Suspect parameter - it is in this context it is important.
  3. Make a script for DredgeAligments.
  4. Start RetroTector and make your subdirectory the current directory.
  5. Execute the DredgeAlignments script. If all is well, you will get a green notice about it.
  6. Take a good look at the GenesCandidate.txt file just generated and see if it makes sense as Genes.txt. If it does, you may rename it to Genes.txt. If it does not, the Dredge.txt file may give a hint about why. Minor flaws may be edited away.
  7. Similarly inspect MotifGroupsCandidate.txt. It is rather sure to contain '??????' instead of a few distance range specifications. Use the editor to substitute something sensible. If you are satisfied, rename it to MotifGroups.txt. 
  8. With the editor, complete MakeSubGenesScript.txt into a valid script for MakeSubGenes. Execute it.
  9. This will have generated a SubGenesCandidate.txt file. Give it too a good critical look and if satisfied, rename it SubGenes.txt. 
  10. Run CheckDatabase on your subdirectory. If its comments are more than you can bear, you will have to try again from somewhere above - most likely 2.
  11. If you get here, add other files as required (see Database for a list). Keep in mind that if you want the files to be identical to those in Ordinary, you need not add them (missing files are automatically picked from Ordinary instead).
  12. Try to run RetroTector with the new Database and some real DNA.
