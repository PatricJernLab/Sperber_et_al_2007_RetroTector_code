RetroTector

 

CollectGenome


This Executor is a subclass of SQLUser which collects Genome databases, as described under SQLUser. While the whole collection is referred to as a 'genome' and its subdivisions as 'chromosomes' CollectGenome can be used on smaller entities than whole genomes.



CollectGenome has five parameters:
  • Generation: The file generation to fetch data from. Default: 1.
  • MinChainScore: Chains with lower score than this are rejected. Default: 250.
  • MinProcessScore: Chains with lower score than this are not given the full time-consuming treatment. The columns RepBaseFinds, PolClass and BestRefRV are not filled unless the job is already done by RetroVID and ORFID (if ClusterMode = Yes). Default: 300.
  • MinLTRPairScore: LTR candidate pairs with lower PairFactor than this and not belonging in a Chain are ignored. Default: 0.
  • MultiPuteins: If Yes, more than one Putein of the same Gene may be associated with the same (multigenus) Chain.
The file generation is contained in the file name. Eg, a file named 002SelectedChains.txt has generation 2.

Below the ordinary contents of a startup parameter window, there are a number of other controls, whose use is described below.


The procedure for using CollectGenome is as follows:

  1. In the Choose genome field, specify the name of a new genome database to create or an old one to extend.
  2. Click 'Add one row'.
  3. Use the file dialog to choose a directory (typically a whole chromosome, possibly zipped) to collect from. Its path will appear above Long comment.
  4. Enter any text of any length that catches your fancy in the 'Long comment' field.
  5. Enter a unique identifier, typically the number of the chromosome, in 'Chromosome'.
  6. Click 'Add row'. The data you just specified will appear as a new row in the chromosome list. If you made an error, it may be removed with Delete last row.
  7. Repeat 2-6 to your heart's content.
  8. Not necessary, but it may be a good idea to select-copy-paste the contents of the chromosome list into a text file. It can then be reimported with the 'Import' button, which can save some tedious work.
  9. Click 'Run' and let CollectGenome work - possibly for days if you are collecting a full genome. The RetroTector main window will give an idea of what is going on. If you chose an old database, you will be asked if you actually want to add to it, and if you want old data belonging to the relevant chromosomes to be removed first. This is useful if a run was interrupted in the middle of a chromosome.
The results may be viewed using any suitable SQL viewer, or Genomeview.

It is also possible (with the 'Import' button) to import a tab-delimited text file into the chromosome list. See point 8 above.
