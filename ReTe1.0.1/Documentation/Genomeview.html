RetroTector

 

Genomeview


This Executor is a subclass of SQLUser and is used to inspect the contents of Genome datbases, as described under SQLUser and collected by CollectGenome.


There are no parameters.


Genomeview uses a set of tabbed panes. At startup only one of these is present. If you are familiar with SQL, it provides the opportunity (now or later) to enter and execute any SQL command and view the result, if any. However, its main function is to specify a genome to view, with the Choose genome  choice box (where the previous choice is marked with an '*'). Once you have done so, the other panes are added (though with a large database and a computer with limited capacity there may be a delay of several minutes):

Chains overview: shows for each chromosome the distribution of Chains along its length. If one of the red lines is clicked, the Chain(s) corresponding to it (and its neighbours) will be shown in the Tables pane. Buttons allow doubling or halving the horizontal scale, and saving of the graphics.

LTRs overview: shows for each chromosome the distribution of LTRs along its length. If one of the blue lines is clicked, the LTR(s) corresponding to it (and its neighbours) will be shown in the Tables pane. Buttons allow doubling or halving the horizontal scale, and saving of the graphics.

Tables: Three scrollable tables show contents of the Chains, Puteins and LTRs tables, either a small selection (after clicking in Chains overview or LTRs overview or another table), a specfied selection, a whole chromosome or the whole genome, as chosen with the controls below each table. Clicking in a table has the following consequences:
  • In a column head: The table is sorted (or resorted in opposite order) on the column in question. If you Shift-Click, the table is double sorted on the column and the most recently clicked column. By repeated Shift-Clicking you may sort on multiple columns, but don't push your luck!
  • In a data cell: if the cell content is textual, it is shown fully in a separate window. In any case, the row is selected, with effects depending on the table:
    • In Chains: The Puteins and LTRs (if any) corresponding to the selected Chain are shown in their tables. The Chain details and Chain info panes receive contents relating to the selected Chain.
    • In Puteins: The Chain to which the selected Putein belongs is shown.
    • In LTRs: The Chain to which the selected LTR belongs (if any) is shown.
It is possible to select multiple lines in a table by platform-dependent modified clicking. The above-mentioned effects on the other table will apply for all the selected items.

Below each table are the following gadgets:
  • A text field showing the total number of rows and number of selected rows.
  • A 'Show where' button. Clicking it changes the table contents defined by the next gadget,
  • A choice box. If it is empty, all elements of the relevant kind are included. If not, it should be a valid SQL 'where' clause, which you may either type yourself, or use one of the provided alternatives to include only one chromosome.
  • An 'Export selected...' button. It exports the selected rows, or all the rows if none are selected, either
    1. to a text file in one of several formats. A list window allows you to choose which columns to export, by reordering and selecting. Tabs, linefeeds and CRs inside the fields will be replaced by semicolons.
    2. to an SQL database in a similar format as that used by CollectGenome, but there is no Chromosomes table. Instead the relevant fields in Chromosomes is added to each of the other three tables.
    3. (at your peril!) to a specified (as database.table) SQL table. A list window allows you to choose which columns to export, by reordering and selecting. New columns are added to the target table as necessary, but other incompatibilities are not handled.
  • A choice box to specify the target of the preceding button. You can select a file format (upper part) a new SQL database (enter name into empty line) or an existing database (lower part) or an SQL table (new or old) by specifying both database and table, separated by a dot.

Chain details: Contains
  • one (or more if the Chain genus is ambiguous) graphic representations of the Chain with its Puteins and LTRs. It should be largely self-explaining. Black boxes denote the longest ORF inside each putein. If a putein box is cross-hatched, it means that the number of stop codons plus the number of shifts exceeds 2. Vertical dashed lines represent ALUs, L1s or other elements excluded by SweepDNA. Buttons below each allow exporting of the graphics to a file, or opening the chain in Chainview, if the original results (possibly zipped) are available; if they are not where expected, you will be asked. Clicking in a Putein opens Puteinview with the putein. Clicking in an LTR opens a window similar to the RepBase similarity pane (see below) relating specifically to the LTR.
  • a similar, but simpler graphic representation of the most similar reference retrovirus in 'forretrotector.refrvs', also with a button for graphics export.
  • a tabbed pane showing graphically regions of similarity (see RepBaseFinds field in SQLUser) between the Chain (horizontal axis) and the consensus sequences from RepBase in 'forretrotector.repbasetemplates' (vertical axis). If the sequence name in the tab is followed by a '<', it means that the find is in the complementary strand.

Chain info: Textual information about the Chain, as described under Chainview.

Ref info: The original HTML page describing the most similar reference retrovirus in 'forretrotector.refrvs'.
